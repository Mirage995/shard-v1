## Key Concepts
- **Bubble Sort**: Repeatedly swaps adjacent elements if they're in wrong order; O(n²) time complexity
- **Selection Sort**: Finds minimum element and places it at beginning; O(n²) time, minimal swaps
- **Insertion Sort**: Builds sorted array one element at a time; efficient for small/nearly-sorted data
- **Merge Sort**: Divide-and-conquer algorithm that splits, sorts, and merges; O(n log n) guaranteed
- **Quick Sort**: Picks pivot and partitions around it; O(n log n) average, O(n²) worst case
- **Heap Sort**: Uses binary heap data structure; O(n log n) time, O(1) space
- **Stability**: Stable sorts preserve relative order of equal elements (important for multi-key sorting)
- **In-place vs Out-of-place**: In-place uses O(1) extra space; out-of-place requires O(n) additional memory

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Simple algorithms (bubble, insertion) easy to understand and implement | Simple algorithms have poor O(n²) performance on large datasets |
| Merge sort guarantees O(n log n) and is stable | Merge sort requires O(n) extra space |
| Quick sort is typically fastest in practice with good cache locality | Quick sort can degrade to O(n²) without proper pivot selection |
| Insertion sort excellent for small or nearly-sorted arrays | Most efficient algorithms are more complex to implement correctly |
| Heap sort has O(n log n) worst-case with O(1) space | Heap sort is unstable and has poor cache performance |

## Practical Example
```python
# Quick Sort Implementation
def quicksort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quicksort(left) + middle + quicksort(right)

# Usage
data = [64, 34, 25, 12, 22, 11, 90]
sorted_data = quicksort(data)
print(sorted_data)  # [11, 12, 22, 25, 34, 64, 90]

# Python's built-in (Timsort - hybrid of merge & insertion)
data.sort()  # In-place
sorted_data = sorted(data)  # Returns new list
```

## SHARD's Take
Sorting algorithms represent a fundamental trade-off between simplicity and efficiency. For production systems, use language built-ins (like Python's Timsort or Java's dual-pivot quicksort) which are highly optimized, but understanding the underlying algorithms is crucial for technical interviews and recognizing when specialized sorting is needed. The "best" algorithm depends entirely on your data characteristics: insertion sort for small/nearly-sorted data, merge sort when stability matters, and quicksort for general-purpose speed with random data.
