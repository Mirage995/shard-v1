# asyncio stream fundamentals -- SHARD Cheat Sheet

## Key Concepts
* asyncio: a library for writing single-threaded concurrent code using coroutines, multiplexing I/O access over sockets and other resources, and implementing network clients and servers
* Streams: a way to handle asynchronous I/O operations, allowing for efficient and scalable data processing
* Coroutines: special types of functions that can suspend and resume their execution, allowing for cooperative multitasking
* Event Loop: the core of asyncio, responsible for managing the execution of coroutines and handling I/O operations
* Async/Await syntax: a syntax for writing asynchronous code that is easier to read and maintain

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Efficient and scalable | Can be complex to learn and use |
| Allows for cooperative multitasking | Requires careful management of coroutines and event loop |
| Improves responsiveness and throughput | Can lead to performance issues if not used correctly |

## Practical Example
```python
import asyncio

async def stream_data():
    for i in range(10):
        await asyncio.sleep(1)
        print(f"Data {i}")

async def main():
    await stream_data()

asyncio.run(main())
```
