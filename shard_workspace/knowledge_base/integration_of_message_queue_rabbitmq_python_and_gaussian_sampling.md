# Integration of message queue rabbitmq python and gaussian sampling -- SHARD Cheat Sheet

## Key Concepts
*   **RabbitMQ:** A message broker that implements the AMQP protocol for asynchronous messaging.
*   **Pika:** A Python client library to interact with RabbitMQ.
*   **Gaussian Sampling:** Generating random numbers from a normal (Gaussian) distribution.
*   **Message Queue:** A buffer that stores messages for asynchronous communication between applications.
*   **Producer:** An application that sends messages to a RabbitMQ exchange.
*   **Consumer:** An application that receives messages from a RabbitMQ queue.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Decouples producer and consumer applications, improving scalability and resilience. | Adds complexity to the system architecture. |
| Enables asynchronous processing of tasks, improving responsiveness. | Requires careful error handling and message acknowledgment to prevent data loss. |
| Gaussian sampling can introduce randomness for simulations or load balancing. | Integrating statistical sampling might not always be relevant for typical message queuing scenarios. |
| Allows for distribution of tasks across multiple workers. | Serialization/deserialization overhead when passing data through the queue. |

## Practical Example
```python
import pika
import random
import time

# RabbitMQ connection parameters
rabbitmq_host = 'localhost'
queue_name = 'gaussian_queue'

# Function to generate a Gaussian sample
def generate_gaussian_sample(mu, sigma):
    return random.gauss(mu, sigma)

# Producer
def producer():
    connection = pika.BlockingConnection(pika.ConnectionParameters(host=rabbitmq_host))
    channel = connection.channel()
    channel.queue_declare(queue=queue_name)

    for i in range(5):
        sample = generate_gaussian_sample(0, 1)  # Mean 0, Standard Deviation 1
        message = str(sample)
        channel.basic_publish(exchange='', routing_key=queue_name, body=message)
        print(f" [x] Sent {message}")
        time.sleep(1)  # Simulate some work

    connection.close()

# Consumer
def consumer():
    connection = pika.BlockingConnection(pika.ConnectionParameters(host=rabbitmq_host))
    channel = connection.channel()
    channel.queue_declare(queue=queue_name)

    def callback(ch, method, properties, body):
        print(f" [x] Received {body.decode()}")

    channel.basic_consume(queue=queue_name, on_message_callback=callback, auto_ack=True)
    print(' [*] Waiting for messages. To exit press CTRL+C')
    channel.start_consuming()

if __name__ == '__main__':
    import threading
    producer_thread = threading.Thread(target=producer)
    consumer_thread = threading.Thread(target=consumer)

    producer_thread.start()
    consumer_thread.start()

    producer_thread.join()
    # Consumer runs indefinitely, so no join for it.
```
