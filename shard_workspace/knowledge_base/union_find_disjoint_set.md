## Key Concepts
*   **Disjoint Set Union (DSU):** A data structure that tracks a set of elements partitioned into a number of disjoint (non-overlapping) subsets.
*   **Find:** Determines which subset a particular element belongs to, typically by finding a representative member of that subset.
*   **Union:** Merges two subsets into a single subset.
*   **Path Compression:** A Find optimization that flattens the structure of the tree by making every visited node point directly to the root.
*   **Union by Rank:** A Union optimization that attaches the shorter tree to the taller tree to minimize tree height.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Efficiently determines connectivity between elements. | Can only represent disjoint sets; cannot represent overlapping sets directly. |
| Supports fast Union and Find operations, especially with path compression and union by rank. |  Not suitable for scenarios requiring frequent splitting of sets. |
| Useful in Kruskal's algorithm for finding minimum spanning trees. | Can be less intuitive to implement compared to other data structures. |

## Practical Example
```python
class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # Path compression
        return self.parent[x]

    def union(self, x, y):
        root_x = self.find(x)
        root_y = self.find(y)
        if root_x != root_y:
            if self.rank[root_x] < self.rank[root_y]:
                self.parent[root_x] = root_y
            elif self.rank[root_x] > self.rank[root_y]:
                self.parent[root_y] = root_x
            else:
                self.parent[root_y] = root_x
                self.rank[root_x] += 1

# Example usage:
dsu = DSU(5)
dsu.union(0, 1)
dsu.union(2, 3)
print(dsu.find(0) == dsu.find(1)) # True
print(dsu.find(0) == dsu.find(2)) # False
