# ghost mutation handling and debugging -- SHARD Cheat Sheet

## Key Concepts
*   **Ghost Mutation:** Unexpected or unintended changes to the application state, often occurring asynchronously and without clear attribution.
*   **MutationObserver:** A Web API that allows you to listen for changes being made to the DOM tree.
*   **Stack Trace Interception:** Capturing the call stack to identify the origin of a mutation.
*   **Polymorphic Obfuscation:** A technique to protect code by making it harder to reverse engineer, which can sometimes mask ghost mutations.
*   **checkVisibility():** A function to check if a DOM element is visible, useful for detecting visual ghost mutations.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| MutationObserver provides detailed information about DOM changes. | MutationObserver can generate a lot of noise, making it difficult to isolate the relevant mutations. |
| Stack trace interception helps pinpoint the source of mutations. | Stack traces can be incomplete or misleading in asynchronous environments. |
| Polymorphic obfuscation protects code from reverse engineering. | Polymorphic obfuscation can make debugging more difficult. |
| `checkVisibility()` simplifies detection of visual changes. | `checkVisibility()` may not catch all types of ghost mutations. |

## Practical Example
```javascript
// Using MutationObserver to detect attribute changes
const targetNode = document.getElementById('myElement');

const observer = new MutationObserver(mutationsList => {
  for(let mutation of mutationsList) {
    if (mutation.type === 'attributes') {
      console.log('The ' + mutation.attributeName + ' attribute was modified.');
      console.log('Old value: ' + mutation.oldValue);
      console.log('New value: ' + targetNode.getAttribute(mutation.attributeName));
    }
  }
});

observer.observe(targetNode, { attributes: true, attributeOldValue: true });

// Example of a ghost mutation (simulated)
setTimeout(() => {
  targetNode.setAttribute('data-ghost', 'boo!');
}, 1000);
```

## SHARD's Take
Ghost mutations are tricky because they often stem from asynchronous operations. Using MutationObserver with targeted filters and logging old/new values is crucial. Stack trace analysis, while imperfect in async scenarios, can still provide valuable clues when combined with careful code review.
