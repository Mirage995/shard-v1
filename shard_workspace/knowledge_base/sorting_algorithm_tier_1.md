# sorting algorithm tier 1 — SHARD Cheat Sheet

## Key Concepts
- **Bubble Sort**: Repeatedly swaps adjacent elements if they're in wrong order; simple but O(n²) time complexity
- **Selection Sort**: Finds minimum element and places it at beginning; also O(n²) but fewer swaps than bubble sort
- **Insertion Sort**: Builds sorted array one element at a time; efficient for small/nearly-sorted data, O(n²) worst case
- **Time Complexity**: Measure of how runtime grows with input size; tier 1 algorithms are generally O(n²)
- **Space Complexity**: All tier 1 sorts are in-place with O(1) extra space
- **Stability**: Property where equal elements maintain their relative order (insertion and bubble are stable)

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Simple to understand and implement | Poor performance on large datasets (O(n²)) |
| Work in-place with minimal memory overhead | Significantly slower than advanced algorithms (merge/quick sort) |
| Insertion sort is fast for small or nearly-sorted arrays | Not suitable for production systems with large data |
| Good for educational purposes and algorithm learning | Bubble sort is rarely used in practice |

## Practical Example
```python
# Insertion Sort - most practical of tier 1 algorithms
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        # Shift elements greater than key one position ahead
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr

# Example usage
data = [64, 34, 25, 12, 22, 11, 90]
sorted_data = insertion_sort(data)
print(sorted_data)  # Output: [11, 12, 22, 25, 34, 64, 90]
```

## SHARD's Take
Tier 1 sorting algorithms are foundational learning tools that teach core algorithmic thinking, but their O(n²) complexity makes them impractical for real-world large-scale applications. Insertion sort stands out as the most useful in practice—it's the go-to choice for small arrays (< 10-20 elements) and is often used as the base case in hybrid algorithms like Timsort. Understanding these algorithms is essential for grasping why more advanced sorting methods exist and when optimization truly matters.
