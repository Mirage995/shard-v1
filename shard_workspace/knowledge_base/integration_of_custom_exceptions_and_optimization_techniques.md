# Integration of custom exceptions and optimization techniques — SHARD Cheat Sheet

## Key Concepts
*   **Custom Exceptions:** User-defined exception classes that extend built-in exception types, allowing for specific error handling.
*   **Exception Handling:** Using `try-catch` blocks to gracefully manage errors and prevent program crashes.
*   **Optimization Techniques:** Methods to improve code performance, such as algorithmic improvements, data structure selection, and caching.
*   **Resource Management:** Ensuring proper allocation and deallocation of resources to prevent leaks and improve efficiency.
*   **Fail-Fast:** A design principle where a system should report any failure immediately.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| **Improved Code Readability:** Custom exceptions make error handling logic clearer. | **Increased Complexity:** Overuse of custom exceptions can clutter the codebase. |
| **Domain-Specific Error Handling:** Tailored exceptions provide more context for errors. | **Maintenance Overhead:** Custom exceptions require additional maintenance. |
| **Centralized Error Management:** Easier to manage and modify error handling logic. | **Potential Performance Impact:** Exception handling can be slower than normal control flow. |
| **Enhanced Debugging:** Specific exception types simplify debugging. | **Risk of Over-Abstraction:** Creating too many exception types can obscure the underlying problem. |
| **Resource Optimization:** Exceptions can signal when resources are exhausted or improperly used, triggering optimization routines. | **Exception Handling Overhead:** Excessive try-catch blocks can degrade performance if not used judiciously. |

## Practical Example
```python
class InsufficientFundsError(Exception):
    """Custom exception for insufficient funds."""
    pass

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError("Insufficient funds to withdraw.")
    return balance - amount

try:
    new_balance = withdraw(100, 200)
    print("New balance:", new_balance)
except InsufficientFundsError as e:
    print("Error:", e)

# Optimization: Memoization (caching) to avoid redundant calculations
cache = {}
def fibonacci(n):
    if n in cache:
        return cache[n]
    if n <= 1:
        return n
    cache[n] = fibonacci(n-1) + fibonacci(n-2)
    return cache[n]

print(fibonacci(10)) # Optimized Fibonacci calculation
```
