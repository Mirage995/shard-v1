# Integration of property-based testing and parameterized queries — SHARD Cheat Sheet

## Key Concepts
*   **Property-Based Testing (PBT):** Testing approach where you define properties that should hold true for a range of inputs, and the testing framework generates inputs to verify these properties.
*   **Parameterized Queries:** SQL queries where parameters are used instead of directly embedding values into the query string, preventing SQL injection.
*   **SQL Injection:** A security vulnerability that allows attackers to inject malicious SQL code into a query, potentially compromising the database.
*   **Test Oracle:** A mechanism for determining whether a test case has passed or failed.
*   **Data Generation:** The process of automatically creating test data for property-based testing.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Prevents SQL injection by ensuring data is treated as data, not code. | Can be complex to set up and define meaningful properties. |
| Improves test coverage by automatically generating a wide range of inputs. | Requires a good understanding of the underlying system and its invariants. |
| Simplifies data access code by using parameters instead of string concatenation. | Performance overhead if not implemented correctly. |
| Facilitates internationalization testing by generating diverse character sets. | May require specialized libraries or frameworks. |

## Practical Example

```python
import hypothesis
from hypothesis import given
from hypothesis.strategies import text, integers
import sqlite3

# Assume a database connection is already established
conn = sqlite3.connect(':memory:')
cursor = conn.cursor()
cursor.execute("CREATE TABLE users (username TEXT, age INTEGER)")

@given(username=text(), age=integers())
def test_insert_user(username, age):
    try:
        # Use parameterized query to prevent SQL injection
        cursor.execute("INSERT INTO users (username, age) VALUES (?, ?)", (username, age))
        conn.commit()

        # Verify the data was inserted correctly
        cursor.execute("SELECT username, age FROM users WHERE username = ?", (username,))
        result = cursor.fetchone()
        assert result is not None
        assert result[0] == username
        assert result[1] == age
    except sqlite3.Error as e:
        # Handle database errors (e.g., unique constraint violations)
        print(f"Database error: {e}")
        conn.rollback()
    finally:
        # Clean up the database after each test
        cursor.execute("DELETE FROM users WHERE username = ?", (username,))
        conn.commit()

# Run the tests (Hypothesis will generate many different usernames and ages)
test_insert_user()

conn.close()
```
