# transformer attention mechanism — SHARD Cheat Sheet

## Key Concepts
* Attention Mechanism: allows models to focus on specific parts of the input when generating outputs
* Transformer Architecture: relies on self-attention mechanisms to handle input sequences in parallel
* Positional Encoding: adds information about the position of each token in the input sequence
* Feedforward Layers: used in Transformers to transform the output of the self-attention mechanism
* Long-Range Dependencies: ability of models to capture relationships between distant parts of the input sequence

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Enables parallelization of input sequences | Requires significant computational resources |
| Effective in capturing long-range dependencies | Can be challenging to interpret and visualize |
| State-of-the-art results in many NLP tasks | May not perform well with very short input sequences |

## Practical Example
```python
import torch
import torch.nn as nn
import torch.optim as optim

class TransformerModel(nn.Module):
    def __init__(self):
        super(TransformerModel, self).__init__()
        self.encoder = nn.TransformerEncoderLayer(d_model=512, nhead=8)
        self.decoder = nn.TransformerDecoderLayer(d_model=512, nhead=8)

    def forward(self, src, tgt):
        encoder_output = self.encoder(src)
        decoder_output = self.decoder(tgt, encoder_output)
        return decoder_output

model = TransformerModel()
```
