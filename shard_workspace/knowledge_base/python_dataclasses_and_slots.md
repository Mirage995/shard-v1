# python dataclasses and slots — SHARD Cheat Sheet

## Key Concepts
*   **Dataclasses:** Automatically generate methods like `__init__`, `__repr__`, etc., based on type annotations.
*   **`__slots__`:**  A class attribute that declares allowed instance attributes, saving memory and potentially improving speed.
*   **Type Hints:** Used by dataclasses to define field types and enable static analysis.
*   **Immutability:** Dataclasses can be frozen to create immutable objects.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| **Dataclasses:** Reduced boilerplate code for data-holding classes. | **Dataclasses:** Can introduce overhead if not used judiciously. |
| **`__slots__`:** Memory savings, especially with many instances. | **`__slots__`:**  Makes instances less flexible; no dynamic attribute assignment. |
| **`__slots__`:** Faster attribute access. | **`__slots__`:** Multiple inheritance with slots requires careful consideration. |

## Practical Example

```python
from dataclasses import dataclass

@dataclass(frozen=True)
class Point:
    x: int
    y: int

    def distance(self, other: 'Point') -> float:
        return ((self.x - other.x)**2 + (self.y - other.y)**2)**0.5

class SimplePoint:
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x = x
        self.y = y
```

## SHARD's Take
Dataclasses provide a convenient way to create data-centric classes, while `__slots__` offer a valuable optimization technique when memory footprint and attribute access speed are critical. Use dataclasses to reduce boilerplate and `__slots__` when you need to optimize memory usage and are willing to sacrifice some flexibility.
