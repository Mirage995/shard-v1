## Key Concepts
*   **Graph Traversal:** Systematically visiting all nodes in a graph.
*   **BFS (Breadth-First Search):** Traverses level by level, using a queue.
*   **DFS (Depth-First Search):** Traverses as deeply as possible along each branch, using a stack (implicitly through recursion).
*   **Queue:** A data structure that follows the FIFO (First-In, First-Out) principle.
*   **Stack:** A data structure that follows the LIFO (Last-In, First-Out) principle.
*   **Visited Set:** A set to keep track of visited nodes to avoid cycles.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| **BFS:** Finds the shortest path in unweighted graphs.  Good for finding nearest neighbors. | **BFS:** Requires more memory as it stores all nodes at a given level. Can be slow for very deep graphs. |
| **DFS:** Requires less memory than BFS.  Useful for detecting cycles and exploring deep paths. | **DFS:** May not find the shortest path. Can get stuck in infinite loops if not implemented carefully. |

## Practical Example
```python
# Example graph represented as an adjacency list
graph = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [],
    'E': ['F'],
    'F': []
}

def bfs(graph, start):
    visited = set()
    queue = [start]
    visited.add(start)
    while queue:
        node = queue.pop(0)
        print(node, end=" ")
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

def dfs(graph, start, visited=None):
    if visited is None:
        visited = set()
    visited.add(start)
    print(start, end=" ")
    for neighbor in graph[start]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)

print("BFS Traversal:")
bfs(graph, 'A')
print("\nDFS Traversal:")
dfs(graph, 'A')
```
