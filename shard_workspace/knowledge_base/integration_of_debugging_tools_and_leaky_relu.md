# Integration of debugging tools and leaky relu -- SHARD Cheat Sheet

## Key Concepts
*   **Leaky ReLU:** An activation function that allows a small, non-zero gradient when the unit is not active, preventing dead neurons.
*   **Debugging Tools:** Software applications used to monitor, analyze, and fix errors in code.
*   **Gradient Vanishing:** A problem in deep learning where gradients become very small, hindering learning. Leaky ReLU helps mitigate this.
*   **Activation Functions:** Functions applied to the output of a neuron to introduce non-linearity in neural networks.
*   **TT-Metalium:** A hardware and software platform for AI acceleration, requiring specialized debugging approaches.

## Pro & Contro
| Pro | Contro |

## Practical Example
```python
import torch
import torch.nn as nn

class LeakyReLUModel(nn.Module):
    def __init__(self, negative_slope=0.01):
        super(LeakyReLUModel, self).__init__()
        self.linear = nn.Linear(10, 1)
        self.leaky_relu = nn.LeakyReLU(negative_slope=negative_slope)

    def forward(self, x):
        x = self.linear(x)
        x = self.leaky_relu(x)
        return x

# Example usage with debugging (using print statements for simplicity)
model = LeakyReLUModel()
input_tensor = torch.randn(1, 10)
output = model(input_tensor)

print("Input:", input_tensor)
print("Output:", output)
print("Linear Output:", model.linear(input_tensor)) # Debugging the linear layer
print("Leaky ReLU Output:", output) # Debugging the Leaky ReLU layer
```
