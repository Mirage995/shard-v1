# chiedo facendo applied to numerical_computation — SHARD Cheat Sheet

## Key Concepts
- **Chiedo Facendo Philosophy**: Learning by doing through direct implementation of mathematical algorithms in code
- **Numerical Computation**: Translating mathematical formulas into executable code to solve real-world problems without analytical solutions
- **Progressive Complexity**: Starting with simple problems (quadratic equations) and advancing to complex systems (differential equations, Monte Carlo)
- **Monte Carlo Method**: Using random sampling to solve deterministic problems through statistical simulation
- **Numerical Integration**: Computing definite integrals via trapezium method, Simpson's rule, or Monte Carlo when analytical solutions fail
- **Matrix Methods**: Solving systems of simultaneous equations using augmented matrices and computational linear algebra
- **Regression Analysis**: Modeling relationships between variables through linear (fitlm) and nonlinear (fitnlm) curve fitting
- **Differential Equation Solvers**: Computational methods for solving rate-of-change equations in dynamic systems
- **Correlation Analysis**: Quantifying linear relationships between continuous variables using product moment correlation

## Pro & Contro
| Pro | Contra |
|-----|--------|
| Concrete implementation builds deeper understanding than pure theory | May neglect numerical stability and error propagation analysis |
| Immediate feedback from running code accelerates learning | Risk of "cookbook programming" without understanding underlying mathematics |
| C provides low-level control; MATLAB enables rapid prototyping | Limited exposure to computational complexity and algorithm efficiency trade-offs |
| Hands-on approach reveals practical limitations of methods | Potential gap between implementation skills and theoretical rigor |
| Applicable to real-world problems from day one | May not emphasize convergence properties and failure mode discovery |

## Practical Example
```matlab
% Monte Carlo Integration: Estimate π by random sampling
n = 1000000;  % number of random points
x = rand(n, 1);
y = rand(n, 1);
inside_circle = (x.^2 + y.^2) <= 1;
pi_estimate = 4 * sum(inside_circle) / n;
fprintf('π estimate: %.6f (error: %.6f)\n', pi_estimate, abs(pi_estimate - pi));

% Linear Regression: Fit data and predict
x_data = [1, 2, 3, 4, 5]';
y_data = [2.1, 3.9, 6.2, 7.8, 10.1]';
model = fitlm(x_data, y_data);
y_pred = predict(model, 6);
fprintf('Prediction at x=6: %.2f\n', y_pred);
```

## SHARD's Take
The chiedo facendo approach to numerical computation is pedagogically optimal because it forces learners to confront the gap between mathematical abstraction and computational reality through direct implementation. However, true mastery requires extending this hands-on philosophy to include experimental discovery of algorithm limitations, numerical instabilities, and convergence behaviors—not just successful implementations. The methodology is strongest when learners write code that deliberately fails, then reason about why.
