# Python Advanced Error Handling — SHARD Cheat Sheet

## Key Concepts
*   **Custom Exceptions:** Create specific exception types for better error identification and handling.
*   **`finally` Block:** Ensures code execution regardless of exceptions, typically for resource cleanup.
*   **Context Managers:** Automate resource management using `with` statements and `__enter__`/`__exit__` methods.
*   **Logging:** Record errors and application behavior for debugging and monitoring.
*   **EAFP (Easier to Ask Forgiveness than Permission):** Handle potential errors directly instead of extensive pre-checks.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Improved code robustness and reliability | Can increase code complexity if overused |
| Easier debugging and maintenance | May require more upfront design effort |
| Graceful error recovery and resource management | Potential performance overhead in some scenarios |
| Enhanced code readability with custom exceptions | Risk of masking underlying issues if not implemented carefully |

## Practical Example
```python
import logging

logging.basicConfig(level=logging.ERROR)

class CustomError(Exception):
    """Base class for custom exceptions."""
    pass

class ValueTooSmallError(CustomError):
    """Raised when the input value is too small."""
    def __init__(self, value, expected):
        self.value = value
        self.expected = expected
        self.message = f"Value {value} is too small. Expected {expected}, got {value}."
        super().__init__(self.message)

def process_value(value):
    try:
        if value < 10:
            raise ValueTooSmallError(value, 10)
        return value * 2
    except ValueTooSmallError as e:
        logging.error(e)
        return None
    finally:
        print("Processing complete.")

result = process_value(5)
print(f"Result: {result}")

result = process_value(12)
print(f"Result: {result}")
```
