# Python Advanced Error Handling -- SHARD Cheat Sheet

## Key Concepts
*   **`try-except` blocks:** Handle potential exceptions during code execution.
*   **`finally` blocks:** Execute code regardless of whether an exception occurred, ensuring cleanup.
*   **Custom Exceptions:** Create specific exception classes for domain-specific error handling.
*   **Context Managers:** Manage resources (files, connections) using `with` statements for automatic cleanup.
*   **Exception Hierarchy:** Organize exceptions into a hierarchy for more granular exception handling.
*   **Exception Chaining:** Preserve the original exception context when raising new exceptions.
*   **Error Handling Decorators:** Wrap functions to automatically handle and log exceptions.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Improved code robustness and reliability | Can add complexity to the code |
| Graceful handling of unexpected errors | Overuse can mask underlying problems |
| Resource management and cleanup | Requires careful planning and implementation |
| Enhanced debugging and error reporting | Can impact performance if not optimized |

## Practical Example
```python
class CustomError(Exception):
    """Base class for custom exceptions."""
    pass

def divide(x, y):
    try:
        result = x / y
    except ZeroDivisionError as e:
        raise CustomError("Division by zero") from e
    else:
        return result
    finally:
        print("Division attempt completed.")

try:
    print(divide(10, 0))
except CustomError as e:
    print(f"Caught CustomError: {e}")
```
