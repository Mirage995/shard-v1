# test driven development pytest — SHARD Cheat Sheet

## Key Concepts
*   **Test-Driven Development (TDD):** A software development process where tests are written before the code.
*   **pytest:** A Python testing framework that simplifies writing and running tests.
*   **Red-Green-Refactor Cycle:** The core TDD process: write a failing test (Red), make it pass (Green), and improve the code (Refactor).
*   **Unit Tests:** Tests that verify the behavior of individual units of code (functions, classes, etc.).
*   **Code Coverage:** A metric that measures the percentage of code executed by tests.
*   **Refactoring:** Improving the internal structure of code without changing its external behavior.
*   **Debugging:** The process of identifying and fixing errors in code.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Improved code quality through early error detection. | Can be time-consuming, especially initially. |
| Clearer requirements and design due to test-first approach. | Requires a shift in mindset and discipline. |
| Reduced debugging time as errors are caught earlier. | Potential for over-testing or testing implementation details. |
| Increased confidence in code changes and refactoring. | Can lead to tightly coupled code if not done carefully. |
| Living documentation of code behavior. |  Steep learning curve for beginners. |

## Practical Example
```python
# test_example.py
import pytest

def add(x, y):
  return x + y

def test_add():
  assert add(2, 3) == 5
  assert add(-1, 1) == 0
  assert add(0, 0) == 0
```

```bash
pytest test_example.py
```

## SHARD's Take
Test-Driven Development (TDD) is a valuable practice, but it's often misunderstood and misapplied. The key is to focus on testing behavior rather than implementation details, and to use tools like pytest and pydantic effectively to streamline the process.
