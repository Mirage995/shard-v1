## Key Concepts

- **Perceptron**: Il primo modello di machine learning (1958), classificatore binario lineare che separa dati in due classi distinte
- **Ispirazione Biologica**: Architettura che imita i neuroni biologici: input (dendriti) → elaborazione (corpo cellulare) → output (assone)
- **Somma Pesata e Bias**: Operazione matematica core: z = (input · pesi) + bias, determina il livello di attivazione del neurone
- **Funzione di Attivazione**: Funzione applicata alla somma pesata (step function, sigmoid) che introduce non-linearità e produce l'output finale
- **Processo di Addestramento**: Apprendimento supervisionato che aggiusta iterativamente pesi e bias basandosi sugli errori di classificazione
- **Separabilità Lineare**: Limitazione fondamentale - può risolvere solo problemi linearmente separabili (non XOR), richiede un iperpiano di separazione
- **Single Layer Perceptron (SLP)**: Architettura con un solo strato computazionale, modella solo confini decisionali lineari
- **Logic Gates**: AND, OR usati come casi di test ideali perché linearmente separabili e con output verificabili

## Pro & Contro

| Pro | Contro |
|-----|--------|
| Semplicità matematica e concettuale, ideale per apprendimento | Limitato a problemi linearmente separabili (non può risolvere XOR) |
| Implementazione da zero richiede poche righe di codice | Architettura shallow inadatta per problemi reali complessi |
| Fondamento storico per comprendere reti neurali moderne | Convergenza non garantita su dati non separabili linearmente |
| Veloce da addestrare su dataset piccoli | Funzione di attivazione step non differenziabile (problemi con gradient descent) |
| Eccellente valore pedagogico per capire gradient descent e weight updates | Sostituito da modelli più sofisticati (MLP, deep learning) nella pratica |

## Practical Example

```python
import numpy as np

class Perceptron:
    def __init__(self, n_inputs, learning_rate=0.1):
        self.weights = np.random.randn(n_inputs)
        self.bias = np.random.randn()
        self.lr = learning_rate
    
    def activation(self, z):
        return 1 if z >= 0 else 0
    
    def predict(self, x):
        z = np.dot(x, self.weights) + self.bias
        return self.activation(z)
    
    def train(self, X, y, epochs=100):
        for epoch in range(epochs):
            for xi, target in zip(X, y):
                prediction = self.predict(xi)
                error = target - prediction
                # Weight update rule
                self.weights += self.lr * error * xi
                self.bias += self.lr * error

# Esempio: Logic Gate AND
X = np.array([[0, 0], [0, 1], [1, 0], [1, 1]])
y = np.array([0, 0, 0, 1])  # AND truth table

perceptron = Perceptron(n_inputs=2)
perceptron.train(X, y, epochs=10)

# Test
for xi in X:
    print(f"Input: {xi} → Output: {perceptron.predict(xi)}")
```

## SHARD's Take

Il Perceptron è un artefatto pedagogico straordinario che demistifica le reti neurali rivelando la loro semplicità matematica fondamentale, ma la sua vera eredità risiede nelle limitazioni che ha esposto piuttosto che nelle capacità dimostrate. Implementarlo da zero è un rito di passaggio essenziale per comprendere backpropagation e decision boundaries, ma i praticanti moderni devono riconoscerlo come un trampolino concettuale verso architetture deep learning, non come uno strumento pratico. La sua incapacità di risolvere XOR ha catalizzato decenni di ricerca che hanno portato alle reti multi-strato, dimostrando che l'intelligenza artificiale richiede gerarchie di astrazione per affrontare la complessità del mondo reale.
