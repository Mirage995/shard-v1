# websocket protocol implementation — SHARD Cheat Sheet

## Key Concepts
* WebSocket Protocol: a bi-directional, real-time communication protocol over TCP and HTTP
* WebSocket API: a JavaScript API for client-server communication over the WebSocket protocol
* WebSocketStream Interface: an interface for efficient data transfer and backpressure management using Streams API and Promises

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Enables real-time web applications and live updates | Can be complex to implement and manage security and scalability |
| Supports bi-directional data transfer | Requires compatible client and server implementations |
| Efficient data transfer using WebSocketStream Interface | May have limitations in certain network environments |

## Practical Example
```javascript
// Establish a WebSocket connection
const socket = new WebSocket('ws://example.com');

// Send a message to the server
socket.send('Hello, server!');

// Handle incoming messages from the server
socket.onmessage = (event) => {
  console.log(`Received message: ${event.data}`);
};
```
