# dynamic programming memoization — SHARD Cheat Sheet

## Key Concepts
*   **Memoization:** Caching the results of expensive function calls and returning the cached result when the same inputs occur again.
*   **Dynamic Programming:** An optimization technique that solves problems by breaking them down into overlapping subproblems and storing their solutions.
*   **Recursion:** A programming technique where a function calls itself to solve smaller instances of the same problem.
*   **Overlapping Subproblems:** The property of a problem where the same subproblems are solved repeatedly during a recursive solution.
*   **Optimal Substructure:** The property of a problem where the optimal solution can be constructed from optimal solutions of its subproblems.
*   **Tabulation:** A bottom-up dynamic programming approach where solutions to subproblems are stored in a table.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Improves performance by avoiding redundant calculations. | Can increase memory usage due to caching. |
| Simplifies code by leveraging recursion. | Can be less efficient than tabulation for some problems due to recursion overhead. |
| Easy to implement for problems with overlapping subproblems. | May not be suitable for problems without overlapping subproblems. |

## Practical Example
```python
def fibonacci_memoization(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fibonacci_memoization(n-1, memo) + fibonacci_memoization(n-2, memo)
    return memo[n]

# Example usage:
print(fibonacci_memoization(10)) # Output: 55
```

## SHARD's Take
Memoization is a powerful technique for optimizing recursive algorithms by storing and reusing the results of expensive function calls. It's particularly effective for problems exhibiting overlapping subproblems, significantly reducing computational complexity. However, it's important to consider the trade-off between improved performance and increased memory usage.
