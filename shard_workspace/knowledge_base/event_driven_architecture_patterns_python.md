# event driven architecture patterns python — SHARD Cheat Sheet

## Key Concepts
*   **Event-Driven Architecture (EDA):** A software architecture pattern where applications react to events.
*   **Message Broker:** A software component that facilitates communication between different services by exchanging messages.
*   **Publisher/Subscriber:** A messaging pattern where publishers send messages to a topic, and subscribers receive messages from that topic.
*   **Asynchronous Communication:** Services communicate without waiting for immediate responses, improving scalability and resilience.
*   **Event Handlers:** Functions or methods that are executed when a specific event occurs.
*   **Domain Events:** Significant occurrences within a domain that domain experts care about.
*   **CQRS (Command Query Responsibility Segregation):** Separates read and write operations for improved performance and scalability.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Decoupled services lead to increased resilience. | Increased complexity in debugging and testing. |
| Improved scalability through asynchronous communication. | Requires robust monitoring and tracing. |
| Enables real-time data processing. | Potential for eventual consistency issues. |
| Promotes modular design and easier maintenance. | Can be harder to reason about than synchronous systems. |

## Practical Example
```python
import asyncio
import aio_pika
import json

async def publish_message(message):
    connection = await aio_pika.connect_robust(
        "amqp://guest:guest@localhost/"
    )
    async with connection:
        channel = await connection.channel()
        await channel.declare_queue("my_queue")
        message_body = json.dumps(message).encode()
        await channel.default_exchange.publish(
            aio_pika.Message(body=message_body),
            routing_key="my_queue"
        )

async def consume_message():
    connection = await aio_pika.connect_robust(
        "amqp://guest:guest@localhost/"
    )
    async with connection:
        channel = await connection.channel()
        queue = await channel.declare_queue("my_queue")
        async with queue.iterator() as queue_iter:
            async for message in queue_iter:
                async with message.process():
                    print(f"Received message: {message.body.decode()}")
                    break

async def main():
    await asyncio.gather(
        publish_message({"event": "user_created", "user_id": 123}),
        consume_message()
    )

if __name__ == "__main__":
    asyncio.run(main())
```

## SHARD's Take
Event-driven architectures are excellent for building scalable and resilient systems, especially microservices. However, the asynchronous nature introduces complexity, requiring careful consideration of error handling, message ordering, and monitoring. Using tools like message queues (RabbitMQ, Kafka) and frameworks like Celery can simplify the implementation.
