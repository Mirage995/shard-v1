# dependency injection python — SHARD Cheat Sheet

## Key Concepts
*   **Dependency Injection (DI):** A design pattern where dependencies are provided to a component instead of the component creating them.
*   **Inversion of Control (IoC):** A principle where the control of object creation and dependency management is inverted to a framework or container.
*   **Coupling:** The degree of interdependence between software modules; DI aims to reduce coupling.
*   **Cohesion:** The degree to which the elements inside a module belong together.
*   **Constructor Injection:** Providing dependencies through the class constructor.
*   **Setter Injection:** Providing dependencies through setter methods.
*   **Method Injection:** Providing dependencies as arguments to specific methods.
*   **Dependency Injector:** A framework or library that automates dependency injection.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Improved Testability | Increased Complexity |
| Better Maintainability | Potential for Over-Engineering |
| Loose Coupling | Steeper Learning Curve |
| Increased Flexibility | Can obscure dependencies if not well-documented |

## Practical Example
```python
class DatabaseConnection:
    def connect(self):
        return "Connected to database"

class UserProfile:
    def __init__(self, db_connection):
        self.db_connection = db_connection

    def get_profile(self, user_id):
        connection_status = self.db_connection.connect()
        return f"Profile for user {user_id} - {connection_status}"

# Constructor Injection
db = DatabaseConnection()
user_profile = UserProfile(db)
print(user_profile.get_profile(123))

## SHARD's Take
Dependency injection is a powerful technique for decoupling components and improving testability, but it can also add complexity to a codebase if not implemented carefully. Python's dynamic nature allows for flexible implementations, but it's important to strike a balance between simplicity and the benefits of DI.
