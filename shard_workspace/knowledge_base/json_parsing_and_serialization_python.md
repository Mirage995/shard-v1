# json parsing and serialization python — SHARD Cheat Sheet

## Key Concepts
*   **JSON (JavaScript Object Notation):** A lightweight data-interchange format that is easy for humans to read and write and easy for machines to parse and generate.
*   **Serialization (Encoding):** Converting Python objects (e.g., dictionaries, lists) into a JSON string.
*   **Deserialization (Decoding):** Converting a JSON string into Python objects.
*   **`json.dumps()`:**  A Python function to serialize Python objects to a JSON formatted string.
*   **`json.loads()`:** A Python function to deserialize a JSON formatted string to Python objects.
*   **`json.dump()`:** A Python function to serialize Python objects and write them to a file.
*   **`json.load()`:** A Python function to read data from a JSON file and deserialize it to Python objects.
*   **Custom Encoding/Decoding:** Implementing custom logic to handle serialization/deserialization of complex Python objects.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Simple and widely supported data format. | Can be verbose compared to binary formats. |
| Human-readable and easy to debug. | Limited data type support compared to Python. |
| Built-in support in Python with the `json` module. | Requires handling of potential `JSONDecodeError` exceptions. |
| Easy to integrate with web APIs. | Custom encoding/decoding can add complexity. |

## Practical Example
```python
import json

# Python dictionary
data = {
    "name": "John Doe",
    "age": 30,
    "city": "New York"
}

# Serialize to JSON string
json_string = json.dumps(data, indent=4) # indent for readability
print("JSON String:", json_string)

# Deserialize from JSON string
loaded_data = json.loads(json_string)
print("Python Dictionary:", loaded_data)

# Writing to a JSON file
with open("data.json", "w") as f:
    json.dump(data, f, indent=4)

# Reading from a JSON file
with open("data.json", "r") as f:
    loaded_from_file = json.load(f)
print("Loaded from file:", loaded_from_file)
```
