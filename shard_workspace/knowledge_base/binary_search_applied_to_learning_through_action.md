# binary search applied to learning through action — SHARD Cheat Sheet

## Key Concepts
- **Binary Search Learning**: Dividing the skill/knowledge space in half repeatedly to find optimal learning paths faster than linear exploration
- **Action-Based Bisection**: Testing the midpoint of your capability range to determine which half needs more practice
- **Feedback Loops**: Using performance results to eliminate half the search space and focus effort efficiently
- **Skill Calibration**: Finding your current competency level by testing progressively harder or easier challenges
- **Adaptive Difficulty**: Adjusting task complexity based on success/failure to maintain optimal learning zone
- **Convergence Speed**: Logarithmic learning efficiency (O(log n)) versus linear trial-and-error (O(n))
- **Boundary Detection**: Identifying the edge between "can do" and "cannot yet do" to focus practice
- **Iterative Refinement**: Each action provides information to narrow the next learning target

## Pro & Contro
| Pro | Contra |
|-----|--------|
| Dramatically faster skill acquisition than random practice | Requires clear success/failure metrics for each action |
| Minimizes time spent on tasks too easy or too hard | Not suitable for skills without ordered difficulty levels |
| Naturally finds your optimal challenge zone (flow state) | Initial setup requires defining the skill spectrum |
| Efficient use of practice time and cognitive resources | May miss foundational skills if jumping too quickly |
| Self-correcting through continuous feedback | Assumes monotonic skill progression (not always true) |
| Scales well to complex skill hierarchies | Requires honest self-assessment of performance |

Binary search transforms learning from wandering exploration into targeted skill acquisition by systematically eliminating half the possibility space with each deliberate action.
