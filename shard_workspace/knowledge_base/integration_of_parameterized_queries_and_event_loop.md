# Integration of parameterized queries and event loop -- SHARD Cheat Sheet

## Key Concepts
*   **Parameterized Queries:** A method of executing SQL queries where values are passed as parameters instead of being directly embedded in the SQL string.
*   **SQL Injection Prevention:** Parameterized queries prevent SQL injection attacks by treating parameters as data, not executable code.
*   **Event Loop:** A programming construct that waits for and dispatches events or messages in a program.
*   **Asynchronous Operations:** Allows the program to continue executing other tasks while waiting for a database query to complete.
*   **Database Drivers:** Libraries that enable communication between the application and the database, often providing support for parameterized queries.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Prevents SQL injection attacks. | Can be more verbose than string formatting. |
| Improves code readability and maintainability. | Requires proper database driver support. |
| Allows for efficient query execution through query plan caching. | Potential performance overhead if not implemented correctly. |
| Enables asynchronous database operations in event loops. | Error handling in asynchronous environments can be complex. |

## Practical Example
```python
import asyncio
import aiosqlite

async def main():
    async with aiosqlite.connect('example.db') as db:
        await db.execute("CREATE TABLE IF NOT EXISTS users (id INTEGER PRIMARY KEY, name TEXT, age INTEGER)")
        await db.execute("INSERT INTO users (name, age) VALUES (?, ?)", ('Alice', 30))
        await db.execute("INSERT INTO users (name, age) VALUES (?, ?)", ('Bob', 25))
        await db.commit()

        async with db.execute("SELECT name, age FROM users WHERE age > ?", (27,)) as cursor:
            async for row in cursor:
                print(f"Name: {row[0]}, Age: {row[1]}")

asyncio.run(main())
```
