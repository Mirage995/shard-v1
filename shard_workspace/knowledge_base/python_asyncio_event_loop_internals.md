# python asyncio event loop internals — SHARD Cheat Sheet

## Key Concepts
*   **Event Loop:** The central execution mechanism that monitors events, schedules callbacks, and executes coroutines.
*   **Coroutine:** A special function that can suspend and resume its execution, enabling concurrent execution.
*   **Callback:** A function that is executed when a specific event occurs.
*   **Task:** A wrapper around a coroutine, managed by the event loop for execution.
*   **Future:** Represents the eventual result of an asynchronous operation.
*   **AbstractEventLoop:** Defines the interface for event loop implementations.
*   **BaseEventLoop:** A concrete implementation of the abstract event loop.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Enables highly concurrent I/O-bound operations. | Can be complex to debug and understand. |
| Improves application responsiveness. | Requires careful management of resources to avoid blocking the event loop. |
| Allows for efficient use of system resources. | Can introduce subtle race conditions if not handled correctly. |

## Practical Example
```python
import asyncio

async def my_coroutine():
    print("Coroutine started")
    await asyncio.sleep(1)
    print("Coroutine finished")
    return "Coroutine result"

async def main():
    loop = asyncio.get_running_loop()
    task = loop.create_task(my_coroutine())
    result = await task
    print(f"Result: {result}")

if __name__ == "__main__":
    asyncio.run(main())
```
