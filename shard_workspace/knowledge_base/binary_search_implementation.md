# binary search implementation -- SHARD Cheat Sheet

## Key Concepts
*   **Divide and Conquer:** Binary search repeatedly divides the search interval in half.
*   **Sorted Data:** Requires the input data to be sorted for correct operation.
*   **Logarithmic Time Complexity:** Achieves O(log n) time complexity, making it efficient for large datasets.
*   **Midpoint Calculation:** Correctly calculating the midpoint is crucial to avoid integer overflow.
*   **Termination Condition:** Defining the correct termination condition is essential to prevent infinite loops.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Efficient for large, sorted datasets | Requires sorted data as input |
| Logarithmic time complexity O(log n) | Not suitable for unsorted data |
| Simple to implement | Can be tricky to get boundary conditions correct |

## Practical Example
```python
def binary_search(arr, target):
    left = 0
    right = len(arr) - 1
    while left <= right:
        mid = left + (right - left) // 2  # Prevent potential overflow
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1  # Target not found

# Example usage:
arr = [2, 5, 7, 8, 11, 12]
target = 13
result = binary_search(arr, target)
if result != -1:
    print(f"Element is present at index {result}")
else:
    print("Element is not present in array")
```
