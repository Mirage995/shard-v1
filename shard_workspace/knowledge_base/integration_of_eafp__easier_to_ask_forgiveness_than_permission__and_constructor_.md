# Integration of eafp (easier to ask forgiveness than permission) and constructor injection — SHARD Cheat Sheet

## Key Concepts
*   **EAFP (Easier to Ask Forgiveness than Permission):** A programming style where you assume something will work and catch exceptions if it doesn't.
*   **Constructor Injection:** Passing dependencies to a class through its constructor.
*   **Dependency Injection:** A design pattern where dependencies are provided to a class instead of the class creating them itself.
*   **Exception Handling:** Using `try...except` blocks to gracefully handle errors.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Cleaner code by avoiding redundant checks. | Can hide potential issues if exceptions are not handled correctly. |
| More efficient in cases where the "normal" path is more common than the exceptional one. | Overuse can make debugging harder. |
| Decouples object creation from dependency resolution. | Can lead to unexpected behavior if dependencies are not properly configured. |

## Practical Example

```python
class DatabaseConnection:
    def __init__(self, db_url):
        self.db_url = db_url
        self.connection = None

    def connect(self):
        try:
            # Attempt to connect to the database (EAFP)
            # In a real scenario, this would involve database-specific code
            # and might raise exceptions if the connection fails.
            print(f"Connecting to {self.db_url}...")
            self.connection = "Successfully connected" #Simulate connection
            return self.connection
        except Exception as e:
            print(f"Failed to connect: {e}")
            self.connection = None
            return None

    def query(self, sql):
        if self.connection:
            try:
                # Attempt to execute the query (EAFP)
                print(f"Executing query: {sql}")
                return "Query result" # Simulate query result
            except Exception as e:
                print(f"Query failed: {e}")
                return None
        else:
            print("No database connection.")
            return None

# Constructor Injection
db_connection = DatabaseConnection("postgres://user:password@host:port/database")

# Using the connection

else:
```
