## Key Concepts
*   **Resource Naming:** Use nouns, not verbs, to represent resources (e.g., `/users` instead of `/getUsers`).
*   **HTTP Methods:** Leverage HTTP methods (GET, POST, PUT, DELETE, PATCH) for CRUD operations.
*   **Status Codes:** Utilize appropriate HTTP status codes to indicate success or failure (e.g., 200 OK, 201 Created, 400 Bad Request, 404 Not Found, 500 Internal Server Error).
*   **Versioning:** Implement API versioning (e.g., `/v1/users`) to maintain backward compatibility.
*   **HATEOAS:** Hypermedia as the Engine of Application State; include links in responses to enable API discovery.
*   **Filtering and Pagination:** Allow clients to filter and paginate large datasets.
*   **Error Handling:** Provide informative error messages in a consistent format (e.g., JSON with error codes and descriptions).
*   **Authentication and Authorization:** Secure the API with appropriate authentication (e.g., OAuth 2.0, API keys) and authorization mechanisms.
*   **Rate Limiting:** Protect the API from abuse by implementing rate limiting.
*   **Idempotency:** Design operations to be idempotent, meaning that multiple identical requests have the same effect as a single request.

## Practical Example

```python
from flask import Flask, jsonify, request

app = Flask(__name__)

users = [
    {'id': 1, 'name': 'Alice'},
    {'id': 2, 'name': 'Bob'}
]

@app.route('/users', methods=['GET'])
def get_users():
    return jsonify(users)

@app.route('/users/<int:user_id>', methods=['GET'])
def get_user(user_id):
    user = next((user for user in users if user['id'] == user_id), None)
    if user:
        return jsonify(user)
    return jsonify({'message': 'User not found'}), 404

@app.route('/users', methods=['POST'])
def create_user():
    data = request.get_json()
    new_user = {'id': len(users) + 1, 'name': data['name']}
    users.append(new_user)
    return jsonify(new_user), 201

if __name__ == '__main__':
    app.run(debug=True)
```
