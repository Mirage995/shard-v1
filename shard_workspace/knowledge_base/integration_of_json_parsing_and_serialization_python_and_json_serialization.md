## Key Concepts
*   **JSON Serialization:** Converting Python objects into JSON strings.
*   **JSON Parsing:** Converting JSON strings into Python objects.
*   **`json.dumps()`:**  Python function to serialize objects to a JSON formatted string.
*   **`json.loads()`:** Python function to parse a JSON formatted string into a Python object.
*   **Custom Encoder:** A class extending `json.JSONEncoder` to handle serialization of custom Python objects.
*   **Custom Decoder:** A class to customize the deserialization process using the `object_hook` parameter in `json.loads()`.
*   **REST APIs:**  Web services that commonly use JSON for request and response data.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Simple data exchange format | Requires handling of data types not natively supported in JSON |
| Human-readable and easy to understand | Can be verbose compared to binary formats |
| Widely supported across languages and platforms | Security vulnerabilities if parsing untrusted JSON data |
| Built-in support in Python | Performance overhead compared to simpler serialization methods |

## Practical Example
```python
import json
from datetime import datetime

class CustomObject:
    def __init__(self, name, created_at):
        self.name = name
        self.created_at = created_at

    def to_dict(self):
        return {'name': self.name, 'created_at': self.created_at.isoformat()}

# Serialization
obj = CustomObject("Example", datetime.now())
json_string = json.dumps(obj.to_dict(), indent=4)
print(f"Serialized JSON: {json_string}")

# Deserialization
def from_json(json_dict):
    return CustomObject(json_dict['name'], datetime.fromisoformat(json_dict['created_at']))

loaded_obj = from_json(json.loads(json_string))
print(f"Deserialized Object Name: {loaded_obj.name}")
print(f"Deserialized Object Created At: {loaded_obj.created_at}")
```
