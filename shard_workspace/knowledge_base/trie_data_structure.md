# trie data structure — SHARD Cheat Sheet

## Key Concepts
*   **Trie (Prefix Tree):** A tree-like data structure used for efficient retrieval of strings based on prefixes.
*   **Node:** Represents a character in a string; each node can have multiple child nodes.
*   **Edge:** The connection between nodes, labeled with a character.
*   **Root Node:** The starting point of the trie, typically representing an empty string.
*   **Prefix Search:** Efficiently finding all strings with a given prefix.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Efficient prefix-based search. | Can be memory-intensive, especially with large alphabets. |
| Supports autocomplete and spell-checking. | Insertion and deletion can be complex. |
| Fast retrieval of words with a common prefix. | Not suitable for all types of data. |

## Practical Example
```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end = True

    def search(self, word):
        node = self.root
        for char in word:
            if char not in node.children:
                return False
            node = node.children[char]
        return node.is_end

    def starts_with(self, prefix):
        node = self.root
        for char in prefix:
            if char not in node.children:
                return False
            node = node.children[char]
        return True

# Example usage
trie = Trie()
trie.insert("apple")
print(trie.search("apple"))  # True
print(trie.search("app"))    # False
print(trie.starts_with("app")) # True
```

## SHARD's Take
Tries excel in prefix-based searches, making them ideal for applications like autocomplete. However, their memory footprint can be a limiting factor, especially when dealing with large datasets or long keys. Therefore, carefully consider the space-time tradeoff before choosing a trie.
