# python thread safety basics -- SHARD Cheat Sheet

## Key Concepts
* Thread Safety: preventing race conditions and ensuring deterministic behavior in concurrent programming
* Locks: synchronization primitives for mutual exclusion and resource protection
* Semaphores: limiting concurrent access to resources
* Events: signaling and thread coordination
* Conditions: conditional waiting and thread synchronization
* Barriers: thread coordination and concurrent execution

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Prevents race conditions | Can introduce performance overhead |
| Ensures deterministic behavior | Requires careful synchronization |
| Enables concurrent execution | Can lead to deadlocks if not managed properly |

## Practical Example
```python
import threading
import time

class Counter:
    def __init__(self):
        self.value = 0
        self.lock = threading.Lock()

    def increment(self):
        with self.lock:
            self.value += 1

def worker(counter):
    for _ in range(10000):
        counter.increment()

counter = Counter()
threads = [threading.Thread(target=worker, args=(counter,)) for _ in range(10)]

for thread in threads:
    thread.start()

for thread in threads:
    thread.join()

print(counter.value)
```
