# UDP socket programming -- SHARD Cheat Sheet

## Key Concepts
* UDP (User Datagram Protocol) is a connectionless protocol that prioritizes speed over reliability
* Socket programming is fundamental for network applications, allowing for communication between devices
* Non-Blocking UDP Socket combines UDP programming with Socket API for high-performance applications
* Packet loss and corruption handling are crucial in UDP socket programming
* Understanding the OSI model and transport layer protocols is essential for effective UDP socket programming

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Faster data transfer due to connectionless nature | No guarantee of packet delivery or order |
| Less overhead compared to TCP | Requires manual handling of packet loss and corruption |
| Suitable for real-time applications | May not be suitable for applications requiring guaranteed delivery |

## Practical Example
```python
import socket

# Create a UDP socket
sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# Bind the socket to a address and port
sock.bind(("localhost", 12345))

# Send data to a remote address
sock.sendto(b"Hello, world!", ("localhost", 12346))

# Receive data from a remote address
data, addr = sock.recvfrom(1024)
print(data.decode())
```
