# binary search tree operations — SHARD Cheat Sheet

## Key Concepts
*   **Binary Search Tree (BST):** A tree data structure where each node has at most two children, with the left child being less than the node and the right child being greater.
*   **Insertion:** Adding a new node to the BST while maintaining the BST property.
*   **Deletion:** Removing a node from the BST while maintaining the BST property.
*   **Search:** Finding a specific node within the BST based on its key value.
*   **In-order Traversal:** Visiting all nodes in ascending order (left, root, right).

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Efficient searching, insertion, and deletion (O(log n) on average). | Can become unbalanced, leading to O(n) worst-case performance. |
| Ordered data retrieval through in-order traversal. | Requires more memory than simple arrays or linked lists. |
| Dynamic data structure, easily adapts to insertions and deletions. | Implementation can be complex, especially deletion. |

## Practical Example
```python
class Node:
    def __init__(self, key):
        self.key = key
        self.left = None
        self.right = None

def insert(root, key):
    if root is None:
        return Node(key)
    if key < root.key:
        root.left = insert(root.left, key)
    else:
        root.right = insert(root.right, key)
    return root

def search(root, key):
    if root is None or root.key == key:
        return root
    if key < root.key:
        return search(root.left, key)
    return search(root.right, key)

# Example usage:
root = insert(None, 50)
root = insert(root, 30)
root = insert(root, 20)
root = insert(root, 40)
root = insert(root, 70)
root = insert(root, 60)
root = insert(root, 80)

result = search(root, 60)
if result:
    print("Found:", result.key)
else:
    print("Not found")

```

## SHARD's Take
Binary search trees offer a good balance between search speed and data organization, but maintaining balance is essential for optimal performance. Understanding the recursive nature of BST operations is key to implementing them correctly.
