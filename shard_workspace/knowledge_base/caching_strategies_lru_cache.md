## Key Concepts
*   **Cache:** A temporary storage area for frequently accessed data to speed up retrieval.
*   **LRU (Least Recently Used):** A caching strategy that evicts the least recently accessed item when the cache is full.
*   **Cache Hit:** When the requested data is found in the cache.
*   **Cache Miss:** When the requested data is not found in the cache and needs to be fetched from the original source.
*   **Time Complexity:** Operations like get and put should ideally be O(1) on average.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Improves performance by reducing latency | Requires memory overhead to store cached data |
| Reduces load on the original data source | Can become inefficient if access patterns are not truly LRU |
| Simple to implement | Requires careful tuning of cache size |

## Practical Example
```python
from collections import OrderedDict

class LRUCache:

    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = OrderedDict()

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        else:
            self.cache.move_to_end(key) # Mark as recently used
            return self.cache[key]

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False) # Remove LRU item

# Example Usage:
lru = LRUCache(2)
lru.put(1, 1)
lru.put(2, 2)
print(lru.get(1))  # Output: 1
lru.put(3, 3)  # Evicts key 2
print(lru.get(2))  # Output: -1 (not found)
lru.put(4, 4)  # Evicts key 1
print(lru.get(1))  # Output: -1 (not found)
print(lru.get(3))  # Output: 3
print(lru.get(4))  # Output: 4
```

## SHARD's Take
LRU cache is a valuable tool for optimizing performance by storing frequently accessed data. Its effectiveness hinges on the principle of locality, where recently accessed items are likely to be accessed again. Choosing an appropriate cache size is crucial to balance memory usage and performance gains.
