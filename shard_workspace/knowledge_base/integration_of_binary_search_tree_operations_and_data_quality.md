# Integration of binary search tree operations and data quality -- SHARD Cheat Sheet

## Key Concepts
*   **Binary Search Tree (BST):** A tree-based data structure where the left subtree of a node contains only nodes with keys less than the node's key, and the right subtree contains only nodes with keys greater than the node's key.
*   **Data Quality:** The overall utility of a dataset as a function of its completeness, validity, consistency and timeliness.
*   **BST Operations:** Fundamental operations like insertion, deletion, search, minimum, maximum, and traversal performed on a BST.
*   **Data Validation:** The process of ensuring that data conforms to defined formats, types, and business rules.
*   **Data Cleansing:** The process of detecting and correcting or removing corrupt or inaccurate records from a dataset.
*   **Balanced BST:** A BST where the height of the left and right subtrees of any node differ by at most one, ensuring O(log n) time complexity for most operations.
*   **Self-Balancing BST:** A BST that automatically adjusts its structure to maintain balance after insertions and deletions (e.g., AVL trees, Red-Black trees).

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Efficient data storage and retrieval (O(log n) for balanced trees). | Can degrade to O(n) performance for unbalanced trees. |
| Natural ordering of data facilitates range queries and sorted output. | Requires careful handling of duplicate data to maintain data quality. |
| BST operations can be integrated with data validation and cleansing routines. | Complex balancing algorithms can be challenging to implement and debug. |
| Suitable for dynamic datasets where insertions and deletions are frequent. | Memory overhead due to tree structure (pointers). |

## Practical Example
```python
class Node:
    def __init__(self, key):
        self.key = key
        self.left = None
        self.right = None

def insert(root, key):
    if root is None:
        return Node(key)
    else:
        if key < root.key:
            root.left = insert(root.left, key)
        else:
            root.right = insert(root.right, key)
    return root

def validate_and_insert(root, key):
    if not isinstance(key, (int, float)):
        print(f"Invalid data type: {type(key)}.  Must be numeric.")
        return root  # Or raise an exception

    if key < 0:
        print(f"Invalid data: {key}. Must be non-negative.")
        return root

    return insert(root, key)

# Example usage
root = None
root = validate_and_insert(root, 5)
root = validate_and_insert(root, 3)
root = validate_and_insert(root, 7)
root = validate_and_insert(root, "invalid") # Data validation prevents insertion
root = validate_and_insert(root, -2) # Data validation prevents insertion
```

## SHARD's Take
Integrating data quality checks directly into BST operations, especially insertion, is crucial for maintaining the integrity of the data stored within the tree. This approach prevents invalid or inconsistent data from entering the structure, ensuring that subsequent operations are performed on a reliable dataset. Balancing the BST is also important to maintain performance.
