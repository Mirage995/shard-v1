## Key Concepts
* Weak reference: a reference that does not increase the reference count of an object, allowing for garbage collection.
* Finalization: a mechanism to ensure resources are cleaned up when an object is garbage collected.
* WeakKeyDictionary: a dictionary that uses weak references for keys, useful for caching and metadata management.
* WeakValueDictionary: a dictionary that uses weak references for values, useful for auto-expiring caches and object tracking.
* WeakSet: a set that uses weak references, useful for event listener management and circular reference prevention.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Efficient memory management | Complexity of garbage collection process |
| Prevention of memory leaks | Need to handle edge cases |
| Useful for caching and metadata management | Requires careful consideration of reference counting |

## Practical Example
```python
import weakref

class ExampleObject:
    def __init__(self, name):
        self.name = name

    def __del__(self):
        print(f"Deleting {self.name}")

obj = ExampleObject("Test")
ref = weakref.ref(obj)

print(ref())  # prints: <__main__.ExampleObject object at 0x...>

del obj

print(ref())  # prints: None
```
