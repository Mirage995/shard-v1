# http client implementation python -- SHARD Cheat Sheet

## Key Concepts
*   **HTTP Request:** A message sent from a client to a server to request a resource or trigger an action.
*   **HTTP Response:** A message sent from a server to a client in response to an HTTP request.
*   **Requests Library:** A simple and elegant Python library for making HTTP requests.
*   **urllib.request:** Python's built-in module for opening URLs, including HTTP and HTTPS.
*   **aiohttp:** An asynchronous HTTP client/server framework for asyncio.
*   **HTTPX:** A next-generation HTTP client for Python that supports both HTTP/1.1 and HTTP/2.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| **Requests:** Simple API, easy to use, widely adopted. | Synchronous, not ideal for high-concurrency applications. |
| **urllib.request:** Built-in, no external dependencies. | Lower-level API, more verbose than Requests. |
| **aiohttp:** Asynchronous, excellent for high-concurrency. | Requires asyncio knowledge, more complex setup. |
| **HTTPX:** Supports HTTP/2 and async, modern API. | Relatively newer, potentially fewer community resources. |

## Practical Example
```python
import requests

try:
    response = requests.get('https://www.example.com')
    response.raise_for_status()  # Raise HTTPError for bad responses (4xx or 5xx)
    print(response.status_code)
    print(response.content)
except requests.exceptions.RequestException as e:
    print(f"An error occurred: {e}")
```
