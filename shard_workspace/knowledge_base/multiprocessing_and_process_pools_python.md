# multiprocessing and process pools python — SHARD Cheat Sheet

## Key Concepts
*   **Multiprocessing:** Utilizing multiple processes to achieve true parallelism, bypassing the GIL limitation.
*   **Process Pool:** A group of worker processes ready to execute tasks, improving efficiency by reusing processes.
*   **Inter-Process Communication (IPC):** Mechanisms for processes to exchange data, such as Queues and Pipes.
*   **GIL (Global Interpreter Lock):** A lock that allows only one thread to hold control of the Python interpreter, limiting true parallelism in multithreaded CPython programs.
*   **Queue:** A thread-safe and process-safe FIFO (First-In, First-Out) data structure for inter-process communication.
*   **Lock:** A synchronization primitive to prevent race conditions when multiple processes access shared resources.

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Achieves true parallelism, bypassing the GIL. | Higher memory overhead due to separate memory spaces. |
| Improved performance for CPU-bound tasks. | More complex inter-process communication. |
| Can utilize all available CPU cores. | Increased overhead for process creation and management. |
| Process Pools simplify parallel task execution. | Debugging can be more challenging. |

## Practical Example
```python
import multiprocessing
import time

def worker(num):
    """Worker function"""
    print(f'Worker {num}: Starting')
    time.sleep(2)
    print(f'Worker {num}: Finishing')
    return num * num

if __name__ == '__main__':
    pool = multiprocessing.Pool(processes=4)
    results = pool.map(worker, range(5))
    pool.close()
    pool.join()
    print('Results:', results)
```

## SHARD's Take
Multiprocessing is essential for CPU-bound tasks in Python, offering true parallelism that threads can't achieve due to the GIL. Process Pools streamline parallel execution, but developers must carefully manage inter-process communication and memory usage to avoid performance bottlenecks and ensure data integrity.
