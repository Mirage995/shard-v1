# network protocol design patterns — SHARD Cheat Sheet

## Key Concepts
*   **Protocol Stack:** Layered architecture for modular protocol design.
*   **Sliding Window:** Flow control mechanism to manage data transmission rate.
*   **Request-Response:** Fundamental interaction model for client-server communication.
*   **Publish-Subscribe:** Asynchronous event-driven communication pattern.
*   **Atomicity:** Ensuring all or nothing execution of operations.
*   **Versioning:** Managing changes and compatibility across protocol versions.
*   **Resource lifetime management:** Handling creation, usage, and destruction of resources.

## Practical Example
**Protocol Stack (Simplified):**

Imagine a simplified HTTP request:

1.  **Application Layer:** Constructs the HTTP request (e.g., `GET /index.html`).
2.  **Transport Layer (TCP):** Establishes a connection, segments the request, and ensures reliable delivery.
3.  **Network Layer (IP):** Routes the packets across the network.
4.  **Data Link Layer (Ethernet):** Transmits the data over a physical link.

Each layer has a specific responsibility, contributing to the overall communication process.

Protocol design patterns are crucial for building robust and maintainable network systems.
