## Key Concepts
*   **RabbitMQ:** A message broker that implements the AMQP protocol.
*   **Message Queue:** A buffer that stores messages for asynchronous communication.
*   **Pika:** A Python client library for interacting with RabbitMQ.
*   **AMQP:** The Advanced Message Queuing Protocol, used for message brokering.
*   **Exchange:** Routes messages to queues based on routing keys and bindings.
*   **Queue:** Stores messages until they are consumed by a consumer.
*   **Binding:** Establishes a connection between an exchange and a queue.
*   **Routing Key:** An attribute of a message used by the exchange to route it to the appropriate queue.
*   **Docker:** A platform for running applications in containers.
*   **Docker Compose:** A tool for defining and managing multi-container Docker applications.

## Pro & Contro
| Pro | Contro |
|-----|--------|

## Practical Example

```python
import pika

# Establish connection
connection = pika.BlockingConnection(pika.ConnectionParameters('localhost'))
channel = connection.channel()

# Declare a queue
channel.queue_declare(queue='hello')

# Publish a message
channel.basic_publish(exchange='', routing_key='hello', body='Hello World!')
print(" [x] Sent 'Hello World!'")

connection.close()
```
