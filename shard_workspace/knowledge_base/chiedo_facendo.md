# chiedo facendo — SHARD Cheat Sheet

## Key Concepts
- **Chiedo facendo** - Italian phrase meaning "I ask by doing" or "asking through action"
- **Learning by experimentation** - Acquiring knowledge through direct engagement rather than passive observation
- **Action-oriented inquiry** - Formulating questions that emerge from hands-on practice and experimentation
- **Iterative discovery** - Refining understanding through cycles of doing, questioning, and adjusting
- **Embodied learning** - Knowledge gained through physical or practical engagement with a subject
- **Fail-forward methodology** - Using mistakes and outcomes from action as primary learning inputs

## Pro & Contro
| Pro | Contro |
|-----|--------|
| Generates authentic, context-specific questions | Can be inefficient without basic foundational knowledge |
| Creates deeper retention through experiential learning | May lead to reinventing solutions that already exist |
| Reveals hidden assumptions and edge cases | Requires tolerance for initial failure and uncertainty |
| Builds practical skills alongside theoretical understanding | Time-intensive compared to direct instruction |
| Increases motivation through active engagement | May develop bad habits if done without reflection |

## Practical Example
```python
# Traditional approach: Read documentation, then code
# Chiedo facendo: Code first, questions emerge from doing

# Start by trying to build something
def process_data(data):
    # Try it - what happens?
    result = data.split()  # Question emerges: What if data isn't a string?
    return result

# Action reveals the question
process_data([1, 2, 3])  # AttributeError!
# Now we ask: "How do I handle different input types?"
# The doing created the meaningful question

# Refined through action
def process_data(data):
    if isinstance(data, str):
        return data.split()
    elif isinstance(data, list):
        return data
    # Each iteration reveals new questions to ask
```
